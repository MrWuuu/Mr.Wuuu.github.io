Solved: 更改 hexo new title 自动生成的文章标题

Posted on 2022-11-07 In Bug
Symbols count in article: 615 Reading time ≈ 1 mins.

Hexo 新建文章后在/path/to/hexo_blog/source/_post目录下生成 Markdown 文件中 Front-matter 的 title 会自动生成与 Markdown 文件名相同的字段。

由于 Hexo 使用 hexo new <title> 在新建文章后生成 Markdown 文件中 Front-matter 的 title 会自动生成与 Markdown 文件名相同的字段，因为文件名具有较严格的格式要求，所以有时会出现Markdown 文件的标题作为发表文章标题不合适的情况，经搜索解决办法后解决问题，记录过程：

Markdown 文件名：’Test’

Markdown 文件中 Front-matter 的 title ：’测试’

查看博客网站，发现文章标题为 ‘测试’，解决问题！

[(28条消息) 个人博客（八）Hexo新建文章、主题小修改_七只小天使的博客-CSDN博客](https://blog.csdn.net/weixin_56904230/article/details/115468748#:~:text=参考： Hexo–文章标题和文件名分离 运行命令 hexo new -s,’welcome’ ‘欢迎’ ，可以创建一个文件名为 welcome 而文章标题是 欢迎 的一个文件。)
